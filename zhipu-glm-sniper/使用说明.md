# 智谱GLM Coding抢购助手 v3.2 使用说明

## 功能概览

### 自动化流程

1. **定时打开页面** — 到点自动打开 `bigmodel.cn/glm-coding`（需安装辅助脚本）
2. **高速监控** — 300ms 间隔 + MutationObserver 实时监听 DOM 变化
3. **自动抢购** — 检测到按钮可购买时自动点击
4. **失败重试** — "抢购人数过多"时自动刷新重试
5. **声音提醒** — 不同状态不同音效，紧急时高频警报

### 安全边界

- **停在确认订单前**，不会自动提交订单
- 所有设置持久化在 Tampermonkey 存储中

## 安装

1. 安装 [Tampermonkey](https://www.tampermonkey.net/)
2. 新建脚本，粘贴 `zhipu-sniper.user.js`，保存
3. （推荐）再新建脚本，粘贴 `auto-open.user.js`，保存
4. 打开 `https://bigmodel.cn/glm-coding`，右上角出现面板

## 使用流程

### 场景一：每天上午 10 点开售（推荐）

1. 打开 `https://bigmodel.cn/glm-coding`
2. 点击面板 **设置**
3. 选择目标套餐（Lite/Pro/Max）
4. 选择计费周期（包月/包季/包年）
5. 保持 **每天固定时间开售** 开启，时间为 `10:00:00`
6. 保存设置
7. **可以关闭页面** — 辅助脚本到点会自动打开
8. 页面会提前 3 分钟打开并自动开始监控
9. 检测到可购买 → 自动点击 + 声音提醒
10. 手动确认订单并支付

### 场景二：不知道开售时间，蹲守

1. 打开 `https://bigmodel.cn/glm-coding`
2. 选择目标套餐
3. 点击 **开始监控**
4. 脚本持续检测按钮状态变化
5. 检测到可购买 → 自动点击 + 声音提醒
6. 手动确认

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+Shift+S` | 开始/停止监控 |
| `Ctrl+Shift+H` | 显示/隐藏面板 |

## 配置说明

| 配置项 | 说明 |
|--------|------|
| 每天固定时间开售 | 默认开启，每天上午 10 点 |
| 提前运行分钟数 | 默认 3 分钟，页面打开后提前开始监控 |
| 单次开售时间 | 关闭每天模式后使用，适合临时指定某一天 |
| 目标套餐 | Lite / Pro / Max |
| 计费周期 | 连续包月 / 连续包季(9折) / 连续包年(8折) |
| 自动点击 | 检测到可买时自动点击按钮 |
| 自动重试 | "抢购人数过多"时自动刷新重试 |
| 监控间隔 | 默认 300ms，越小越快 |
| 到点自动刷新 | 开售时刷新页面获取最新状态 |
| 提前刷新秒数 | 开售前几秒刷新（默认3秒） |

## 按钮状态说明

| 状态 | 含义 | 脚本行为 |
|------|------|----------|
| 🟢 可购买 | 按钮可点击 | 自动点击（如开启） |
| 🟡 抢购中 | "抢购人数过多，请刷新再试" | 自动重试刷新 |
| 🔴 未开放 | 按钮禁用 | 持续监控等待 |
| ⚫ 已售罄 | 售罄/已结束 | 停止监控 |

## 技术细节

- **MutationObserver**：监听 DOM 变化，比定时轮询更快
- **多策略按钮查找**：按选择器、文本关键词、可见性多维度匹配
- **智能卡片定位**：通过价格元素和套餐名定位 Lite/Pro/Max 卡片
- **计费周期切换**：自动切换 连续包月/包季/包年 标签页
- **配置持久化**：设置保存在 Tampermonkey 存储中

## v3.1 优化

- 修复套餐提示可能显示为 `undefined` 的问题
- 修复到点自动刷新在开售后反复刷新的风险
- 自动点击成功后停止监控，避免重复点击购买按钮
- "抢购人数过多"时按重试间隔刷新，避免空转监控

## v3.2 优化

- 默认改为每天 `10:00:00` 开售
- 默认提前 `3` 分钟打开页面并启动监控
- 设置面板新增“每天固定时间开售”“每天开售时间”“提前运行分钟数”

## 注意事项

- 脚本仅做监控和辅助点击，最终确认需手动操作
- 建议提前登录账号
- 网络环境要稳定
- 首次使用建议在非开售时段测试
- 如果页面结构变化导致检测不到按钮，可能需要更新选择器
